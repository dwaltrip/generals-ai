# Strakam Paper Summary

**Date:** 2026.05.02
**Source:** Straka & Schmid, "Artificial Generals Intelligence: Mastering Generals.io with Reinforcement Learning," arXiv:2507.06825v2 (10 Jul 2025).
**Purpose:** Capture the load-bearing facts from the paper as a standalone reference. No project-specific takeaways here — those live in the companion takeaways doc.

This doc covers what the paper actually says. Things the paper *doesn't* say are flagged in §8.

---

## 1. TL;DR

- **Two-stage training:** BC on filtered expert replays (3h on H100) → PPO self-play with potential-based reward shaping and an argmax opponent pool of size N=3 (36h on H100).
- **Architecture:** U-Net torso (same as DeepNash / Perolat et al. 2022) + value head + policy head outputting `H×W×9` per-cell action logits.
- **Memory:** *hand-crafted* memory features — four feature-map categories. No recurrent or transformer memory. The paper explicitly lists learned memory as future work.
- **BC corpus:** 347k raw replays → filtered to 16,320 games via patch + length + skill (≥70-star rating) filters.
- **Reward shaping is qualitative-behavior-changing, not just sample-efficiency-changing.** Without it, self-play converges to "overly aggressive, exploitable" play. With it, the agent shifts to building material advantage. Shaping is potential-based on a weighted log-ratio of land/army/castle counts.
- **Result:** the final agent (zero v3) reaches top 0.003% on the 1v1 ladder, ~25th place among humans; 54.8% win-rate vs. Human.exe (the prior SOTA, hand-crafted heuristic agent).
- **Each stage in the recipe matters:** ablation table shows BC < +SP < +shaping < +population, with each step a meaningful win-rate gain.

## 2. Game and environment

Most of this is well-known but a few details the paper makes explicit:

- **Grid generation:** ~20% mountains; 9–11 castles per game, each randomly initialized with 40–50 neutral units; generals placed ≥15 BFS steps apart; each player guaranteed at least one castle within BFS radius 6.
- **Fog of war:** Moore (8-neighbor) visibility around owned cells. Plus a **global scoreboard** always showing each player's land + army count totals. The scoreboard carries information through fog and is what humans use for game-state reasoning under uncertainty.
- **Action format:** `[pass, i, j, direction, split]`. `split=0` moves all-but-one unit; `split=1` moves exactly half. Note this is the *output* format. The policy head's *internal* representation is per-cell (see §4.3).
- **Default reward:** sparse ±1 win/lose. Shaped reward is opt-in — utilities exist for it but it's not baked into `env.step`.
- **Turns and timing:** Turns occur once per second, and consist of two half-turns spaced 500ms apart. Players can make a single move each half-turn (2 moves per turn), which then execute simultaneously. Move conflicts are handled via a set of priority rules.

## 3. Behavior cloning

### 3.1 Corpus and filtering

- **Raw:** 347,000 replays from the Generals.io AWS bucket.
- **Filters applied (in order):**
  1. **Length cutoff:** drop games >500 turns (=1000 moves). The paper observes that "long games are strongly associated with atypical, very often suboptimal behavior of players."
  2. **Patch filter:** keep only games on the newest patch.
  3. **Skill filter:** require at least one player with ≥70-star rating.
- **Final dataset size:** 16,320 games.
- **Quality > quantity:** explicitly stated — "agents perform better when trained on high-quality replays of high-rated players (compared to a larger dataset that includes lower-quality games)."

### 3.2 Training

- **3 hours wall-clock on a single H100.**
- **Loss:** the paper just says "predicting the next move of human players." It doesn't specify whether the loss is joint cross-entropy over the full action vector, factored across action components, or something else.
- **No data augmentation mentioned.** Whether they used board symmetries (4× rotations/flips) is not stated.

### 3.3 BC alone is decent but exploitable

> "[BC alone] is already powerful enough to beat mediocre players, but is very narrow in its behavior and can be easily exploited."

In the bot Elo ladder (§7.2), BC alone scores 1874 — comfortably beating Flobot (1500) and HASP (1710), but well below Human.exe (2018) and the final agent (2052).

## 4. Architecture

Section 5 of the paper.

### 4.1 Backbone: U-Net (DeepNash lineage)

The backbone is a U-Net torso. The paper is explicit: *"We use the same architecture as Perolat et al. (2022)"* — i.e., the DeepNash agent for Stratego.

The reason for U-Net over a plain ResNet is implicit but consistent with the design pattern: encoder-decoder with skip connections gives both global context (via downsampling) and local precision (via upsampling + skips), which is well-suited to per-cell action outputs on grid games.

### 4.2 Hand-crafted memory features

The paper augments observations with **four explicit memory feature-maps** before feeding into the network. This is in lieu of any recurrent or transformer memory.

The four categories:

1. **Revealed castles/generals:** positions of any castles or generals that have been revealed.
2. **Cells the agent has explored:** so it doesn't re-search the same ground.
3. **Cells the agent knows the opponent has seen:** signals where the opponent has scouted, and indirectly marks territory behind fog that the enemy owns.
4. **Last 7 moves taken by each side:** short action history.

The paper explicitly acknowledges the limitations:

> "[T]his hand-crafted memory augmentation does not capture all the details that an agent should remember. For example, the agent does not know *when* it uncovered certain parts of the map, or what were the exact army counts of these cells at that time. We anticipate that incorporating a recurrent architecture or transformer-style memory would allow the network to aggregate and remember those counts and timings, as well as understand that this information is not completely reliable."

In other words, even the SOTA 1v1 agent uses purely feedforward + hand-crafted memory.

### 4.3 Heads

- **Policy head:** outputs an `H×W×9` tensor encoding the action distribution. The 9 channels per cell are: `pass` + 4 directions × 2 splits = 9. Per-cell, fully convolutional. Total action space is `H × W × 9` with masking to enforce validity (owned + army > 1 + dest passable).
- **Value head:** scalar value, predicted from the U-Net's spatial embedding.
- The paper describes only policy + value heads; no auxiliary heads are mentioned.

### 4.4 Input channels

The paper does not give the exact `C` for the observation tensor `R^(C×H×W)`. The repo's environment-level observation has 14 channels; memory features add more. Best estimate is on the order of 20–25 input channels in the trained network. Ground truth is in the actual training code, not the paper.

## 5. Self-play / PPO

### 5.1 Algorithm

- **PPO** (Schulman 2017), with **GAE** (λ=0.95) for stable advantage estimation given long episodes.
- Standard clipped objective + value loss + entropy bonus assumed (paper doesn't go into detail).

### 5.2 Opponent pool

- **N=3 most recent predecessors** in the pool.
- **Opponents play via argmax** (deterministic), not stochastic sampling. The reasoning:

> "we observed that training against stochastic opponents makes new agents focus on exploiting mistakes caused by stochasticity, instead of focusing on more robust gameplay."

- **Replacement criterion:** when a new candidate hits **45% win-rate vs. the current pool** (which corresponds to roughly 55% in argmax-vs-argmax play, since stochasticity hurts the new candidate), it replaces the oldest model in the pool.
- The agent is presumably stochastic during training (otherwise PPO has no exploration); it's the *opponents* that are deterministic.

### 5.3 Compute

- **36 hours on a single H100** for the final agent (zero v3).

## 6. Reward shaping

### 6.1 Form

Standard Ng et al. (1999) potential-based shaping:

```
r_shaped(s, a, s') = r_original(s, a, s') + γ · φ(s') − φ(s)
```

This formulation has the property that the optimal policy set is invariant — shaping doesn't change which policy is best, only the gradient signal density.

### 6.2 Potential function

```
φ(s) = 0.3 · φ_land(s) + 0.3 · φ_army(s) + 0.4 · φ_castle(s)
```

Each sub-potential is a normalized log-ratio of agent vs. enemy counts:

```
φ_x(s) = log(x_agent(s) / x_enemy(s)) / log(max_ratio)   for x ∈ {land, army, castle}
```

Notes:

- **Log-ratio** makes the potential symmetric around equality (ratio = 1 → φ_x = 0).
- **max_ratio** is a normalization constant bounding each component to [-1, 1].
- **Castle weight (0.4) > land/army (0.3 each):** mild prior toward castle play.

### 6.3 Effect (qualitative)

> "Without reward shaping, the agent quickly converges to an aggressive behavior, where most of the time it focuses entirely on gathering army and attacking, which can be fended off by experienced players who defend well. With reward shaping, the agent's focus shifts toward building material advantage and avoiding unnecessary risks, making it much more robust in all stages of the game."

This is the strongest statement in the paper that shaping is *qualitative-behavior-changing*, not just sample-efficiency-changing. Without shaping, the policy converges to a different equilibrium (aggressive but exploitable). The "potential-based shaping preserves optimal policy set" theorem doesn't help here because PPO doesn't find the global optimum — shaping tilts which local optimum it finds.

## 7. Results and emergent behaviors

### 7.1 Headline numbers

- **zero v3 (final agent):** top 0.003% of 1v1 leaderboard (~25th among humans), 4,700 ladder games as nicknames "Average Joe" and "zero v3."
- **vs. top 5 humans:** ~10% win rate (still meaningfully below the top tier).
- **vs. Human.exe (prior SOTA bot):** 54.8% win-rate over 529 games (95% Wilson CI: 50.6%–59.0%).

### 7.2 Bot Elo ladder

| Agent | Elo |
|---|---|
| zero v3 (paper) | 2052 |
| Human.exe (prior SOTA, hand-crafted) | 2018 |
| BC alone (paper, before self-play) | 1874 |
| HASP (Xu 2018) | 1710 |
| Flobot (Bhatia 2023) | 1500 (anchor) |

BC alone is +374 Elo over Flobot, comfortably beats HASP, lands ~150 Elo below Human.exe.

### 7.3 Ablation table

Pairwise win-rates (row vs. column) over 2,000 games each:

|  | BC | +SP | +SP+RS | +SP+RS+Pop |
|---|---|---|---|---|
| BC | — | 31.2% | 26.4% | 26.4% |
| +SP | 68.8% | — | 28.1% | 21.6% |
| +SP+RS | 73.6% | 71.9% | — | 29.8% |
| +SP+RS+Pop | 73.6% | 78.4% | 70.2% | — |

Each stage adds meaningful win-rate. Reward shaping is the largest single jump after BC→SP. Population (presumably the N=3 pool, vs. naive single-opponent self-play) adds another layer.

### 7.4 Emergent behaviors (qualitative)

- **Feints and sidesteps:** unexpected movements to outmaneuver opponents. The paper claims movements "not seen even by the top players."
- **Snowballing:** timely attacks just before the 25-turn land tick, "intermezzo" moves that prepare for later, whole-board awareness.
- **Backdooring:** creating "islands" of cells inside enemy territory for surprise attacks.

### 7.5 Failure modes

- **Mountain dead-ends:** agent gets stuck. The paper diagnoses this as the CNN's inability to compute distances — it "thinks" it can pass through impassable regions.
- **Mode-locked behavior:** agent gets fixated on one of {attack, defense, castle taking} and ignores the others. Specific replay shows the agent capturing castles while being attacked.

## 8. What the paper doesn't tell us

Things one would want to know that aren't in the paper. Mostly answerable by reading the strakam codebase or the DeepNash supplementary material.

- **U-Net depth, channel widths, total parameter count.** "Same architecture as Perolat 2022" is the answer; specifics are in DeepNash's supplementary material.
- **Exact input channel count** for the trained network with memory features included. Repo has 14 base obs channels; memory adds more; total likely 20–25. Ground truth is in the training code.
- **BC loss formulation.** Joint cross-entropy over the full action vector? Factored across action components? Action masking inside the loss or only at sampling time?
- **BC validation metrics.** They don't report top-k action accuracy or any other intermediate signal, just final game performance.
- **Optimizer / learning rate / batch size for BC and PPO.** Standard PPO defaults presumed but unconfirmed.
- **Data augmentation.** Whether they used board symmetries (4× via rotations/flips). Almost free if they didn't.
- **Network normalization scheme.** BatchNorm, LayerNorm, GroupNorm — not stated.
- **Training-data filtering edge cases.** What if a game has both a 70-star and a 30-star player? Train only on the 70-star's perspective? On both? The paper says "at least one participant ≥70 stars" qualifies the *game* for inclusion, but doesn't specify which player's perspective is used for supervision.

## 9. Pointers for follow-up reading

If/when more architectural detail is needed than this paper provides:

- **Perolat et al. 2022 (DeepNash, Stratego):** the cited architecture source. Supplementary material likely has the full network spec. arXiv: 2206.15378 (Science version: doi 10.1126/science.add4679).
- **Strakam codebase, training files:**
  - `examples/_experimental/ppo/train2.py` and `train.py` — published training code. Note from the repo survey that this is *not* the paper's actual training code (no self-play, hardcoded random opponents). Useful as a JAX/Equinox idiom reference but not as a recipe.
  - `examples/_experimental/ppo/network.py` — example network. Drops 5 of 14 obs channels and uses a 9-channel policy head; reasonable JAX/Equinox reference, not the paper's actual network.
- **`remote/generalsio_state.py`** — live-protocol decoder, useful for both replay parsing and live deployment.

### Paper's own future work directions

The paper's §7 explicitly flags these as planned next steps:

- **Learned memory:** recurrent or transformer-style memory to replace the hand-crafted feature maps (also discussed in §4.2 above).
- **Graph neural networks:** as an alternative architecture, motivated by the game's inherent graph structure (grid with impassable mountains makes topology non-trivial).
- **Game-theoretic approaches:** to reduce exploitability of the learned policy.
- **Multi-team and free-for-all modes:** extending the environment and agent beyond 1v1.
- **JAX migration:** for accelerator-agnostic, high-performance training.
