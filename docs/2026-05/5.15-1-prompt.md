**Found and fixed one viewer bug**: annotations (event-log entry, general→city icon flip, leaderboard-dead state, skull) were appearing one step early. Root cause was a snapshot-vs-event alignment confusion — `SNAPS[i]` is the *pre-resolution* state of sim-timestep `i`, but an event with `e.t = T` is the action that takes `SNAPS[T] → SNAPS[T+1]`. Fix: tightened the JS comparisons to strict `<` and shifted the synthesized win event's `t` down by one. **No simulator bug** — the sim's timing is correct; the viewer's annotation rules were off-by-one against the snapshot semantics.
